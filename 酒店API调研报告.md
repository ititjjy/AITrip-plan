# 酒店查询 API 调研报告

> 调研日期：2026-03-24

---

## 一、各 API 详细调研

### 1. Booking.com Affiliate / Demand API

| 项目 | 详情 |
|------|------|
| **类型** | 官方合作伙伴 API |
| **注册方式** | 需要申请 Affiliate Partner 或 Demand API 合作伙伴资格 |
| **是否需要信用卡** | 不需要信用卡，但需要**商业审核** |
| **免费层** | 无公开的免费层；需通过审核成为合作伙伴后才能使用 |
| **调用限制** | 取决于合作伙伴等级，无公开的每月调用限额 |
| **审核难度** | **高** — 需要有已上线的网站/应用，证明有真实流量和商业用途 |
| **返回数据** | 酒店名称、地址、评分、图片、设施、房型、实时价格 |
| **预订功能** | 支持（通过 affiliate 链接或 Demand API 直接预订） |
| **中国城市支持** | 支持全球城市，包括中国 |
| **测试数据** | 无公开的测试/沙盒环境 |

**结论**：Booking.com 官方 API **不适合个人开发者或 Demo 项目**。申请门槛高，需要有真实的商业网站和流量，审核周期长，不提供免费试用。

### 2. RapidAPI 上的酒店 API

RapidAPI 平台上有多个第三方提供的 Booking.com 数据 API，以下是主要选项：

#### 2a. Booking API (by ApiDojo)
| 项目 | 详情 |
|------|------|
| **地址** | https://rapidapi.com/apidojo/api/booking |
| **免费层** | Basic 计划 $0/月，约 **500 次请求/月** |
| **是否需要信用卡** | RapidAPI 注册不需要信用卡（Basic 计划） |
| **返回数据** | 酒店列表、房间价格、设施、评分、图片、位置 |
| **中国城市支持** | 支持（基于 Booking.com 数据） |
| **预订功能** | 不支持直接预订，仅查询 |
| **数据实时性** | 实时抓取 Booking.com 数据 |
| **稳定性** | 中等 — 第三方爬虫 API，可能不稳定 |

#### 2b. Booking.com API V2 (by georgekhananaev)
| 项目 | 详情 |
|------|------|
| **地址** | https://rapidapi.com/georgekhananaev/api/booking-com-api-v2 |
| **免费层** | 无免费层，起价 $299/月 |
| **结论** | **不适合**，价格过高 |

#### 2c. Booking COM API (by LnTtechnologies)
| 项目 | 详情 |
|------|------|
| **地址** | https://rapidapi.com/LnTtechnologies/api/booking-com-api3 |
| **免费层** | Basic 计划 $0/月 |
| **付费计划** | Pro $8/月 |
| **结论** | 有免费层，但调用次数有限 |

**RapidAPI 总结**：ApiDojo 的 Booking API 是最佳选择，提供 **500次/月免费调用**，无需信用卡，数据较完整。但作为第三方爬虫 API，**稳定性和长期可用性无法保证**。

### 3. Amadeus Hotel API (Self-Service)

| 项目 | 详情 |
|------|------|
| **类型** | 全球分销系统 (GDS) 官方自助 API |
| **注册方式** | 在线自助注册，即时获取 API Key |
| **是否需要信用卡** | **不需要信用卡** |
| **免费层** | 测试环境完全免费，每月有免费调用配额（各端点不同，通常数百次） |
| **生产环境** | 每月有一定免费配额，超出部分按量计费（约 $0.01-0.04/次） |
| **测试数据** | 提供**专门的测试环境**，使用有限但真实的测试数据 |

#### 可用的酒店 API 端点：

| 端点 | 功能 | 说明 |
|------|------|------|
| **Hotel List** | 按城市/坐标搜索酒店列表 | 返回酒店名称、位置、hotelId |
| **Hotel Search** | 查询可用房间和实时价格 | 需要 hotelIds + 入住/退房日期 |
| **Hotel Booking** | 预订酒店 | 需要 offerId + 住客信息 + 支付信息 |
| **Hotel Ratings** | 情感分析评分 | 基于用户评价的情感分析 |

#### 返回的数据字段：
- 酒店名称、位置（经纬度）
- 房间类型和描述
- 实时价格和货币
- 支付政策
- 预订确认 ID

#### 局限性：
- **搜索方式有限**：仅支持按 IATA 城市代码或经纬度搜索，**不支持中文城市名直接搜索**
- **测试数据有限**：测试环境数据覆盖不全，某些城市可能无数据
- **中国城市**：理论上支持（通过 IATA 代码如 BJS=北京、SHA=上海），但测试环境中中国城市数据可能不完整
- **数据质量问题**：有用户反馈存在重复记录和缺失地理坐标的问题
- **不返回酒店图片**：Hotel List 和 Hotel Search 端点不返回酒店图片

**结论**：Amadeus 是**最正规、最容易获取的免费酒店 API**。注册简单，无需信用卡，有测试环境。但数据字段不够丰富（缺少图片、设施详情），且对中国城市的支持可能不完善。

### 4. SerpApi Google Hotels API

| 项目 | 详情 |
|------|------|
| **类型** | Google Hotels 搜索结果爬虫 API |
| **注册方式** | 在线注册 |
| **是否需要信用卡** | 免费计划**不需要信用卡** |
| **免费层** | **250 次搜索/月**，50 次/小时吞吐量限制 |
| **付费计划** | 从 $50/月起 |

#### 返回的数据字段：
- 酒店名称 (name)
- GPS 坐标 (gps_coordinates)
- 总体评分 (overall_rating)
- 价格信息
- 品牌信息 (brands)
- 设施/服务 (amenities)
- 广告信息 (ads)
- 分页支持 (serpapi_pagination)

#### 搜索参数：
- `q` — 搜索关键词（支持中文，如"北京酒店"）
- `check_in_date` / `check_out_date` — 入住/退房日期
- `gl` — 国家代码（如 cn）
- `hl` — 语言代码（如 zh-CN）
- `currency` — 货币代码（如 CNY）

#### 优势：
- **完美支持中文城市** — 直接用中文搜索
- **数据丰富** — 来自 Google Hotels 的聚合数据，包含多个 OTA 的价格对比
- 支持按日期查询价格
- 返回评分、图片链接等

#### 局限性：
- **不支持预订** — 仅搜索和价格比较
- 250 次/月的免费额度较低
- 本质上是搜索引擎爬虫，不是官方 API

**结论**：SerpApi 是**数据最丰富、对中文支持最好**的选项。250次/月的免费额度对 Demo 够用。缺点是不支持预订。

## 三、推荐方案

### 方案 A（推荐）：SerpApi + 模拟数据混合方案

**适用场景**：行程规划 Demo，需要真实感的酒店数据展示

1. **搜索和展示**：使用 SerpApi Google Hotels API（250次/月免费）
   - 支持中文城市名直接搜索
   - 返回酒店名称、评分、价格、图片
   - 数据来自 Google Hotels 聚合，质量高

2. **预订功能**：使用模拟数据
   - SerpApi 不支持预订，预订流程用模拟实现
   - 用户点击预订时生成模拟确认号

3. **超出免费额度时**：降级到本地缓存/模拟数据

### 方案 B：Amadeus + SerpApi 组合

**适用场景**：需要真实预订流程的 Demo

1. **搜索**：SerpApi 负责搜索展示（中文支持好）
2. **价格和预订**：Amadeus 负责实时价格查询和预订（正规 GDS 数据）
3. **注意**：两个 API 的酒店 ID 系统不同，需要做映射或分别调用

### 方案 C（最简单）：纯模拟数据方案

**适用场景**：纯展示型 Demo，无需真实数据

完全使用本地 JSON 模拟数据，好处是：
- 零成本，无 API 限制
- 完全可控的数据质量
- 可以包含所有想要的字段

## 五、最终建议

### 对于行程规划 Demo 项目，推荐分阶段实施：

**第一阶段 — 快速搭建（模拟数据）**：
- 先用本地模拟数据把完整的酒店搜索、展示、预订 UI 和流程做完
- 保证所有数据字段齐全，UI 效果完整

**第二阶段 — 接入真实 API**：
- 注册 SerpApi（免费 250次/月），替换酒店搜索功能
- 注册 Amadeus（免费，无需信用卡），作为备选数据源
- 实现 API 调用失败时降级到模拟数据的容错机制

**第三阶段 — 优化**：
- 添加 API 响应缓存，减少调用次数
- 对热门城市预缓存数据
- 实现智能降级策略

### 关键提醒：
1. **没有完全免费且无限制的酒店 API** — 所有可用选项都有调用次数限制
2. **模拟数据是 Demo 项目的最佳起点** — 可以完全控制数据质量和字段完整性
3. **SerpApi 是最推荐的真实 API** — 中文支持好，注册简单，免费额度够 Demo 使用
4. **Amadeus 是最正规的选择** — 但对中文支持和数据丰富度不如 SerpApi
5. **RapidAPI 上的第三方 API 有风险** — 稳定性和合规性无法保证
